# Final Project Report - “Система бронювання в готелі”

## Команда “Hello Bugs” 

## Основний опис

**Система бронювання в готелі** - це повноцінна веб-платформа управління бронюваннями в готелі, яка була розроблена в рамках курсу “Методи розробки програмних систем” НаУКМА в 2025 році. 

***Мета системи - автоматизувати процес бронювання, забезпечити комфортний користувацький досвід та зручність у роботі персоналу готелю***

Система підтримує **4 рівні доступу**:
- **Нереєстрований гість**
- **Зареєстрований гість (GUEST)**
- **Персонал (STAFF)**
- **Адміністратор (ADMIN)**
Система дозволяє гостям онлайн бронювати номери у готелі, а персоналу (в залежності від ролі) - керувати компонентами системи:

**Зареєстровані клієнти** можуть переглядати доступні номери, обирати дати, сортувати і фільтрувати номери за параметрами, створювати бронювання (дані користувача автоматично підтягуються з профілю ), редагувати/скасовувати свої бронювання, отримувати підтвердження та скасування бронювання електронною поштою, переглядати відгуки, залишати відгук, переглядати контакти готелю, переглядати профіль та мати можливість змінити особисту інформацію чи пароль без повторного входу.

**Нереєстровані клієнти** можуть переглядати доступні номери, обирати дати, сортувати і фільтрувати номери за параметрами, створювати бронювання з уведенням свої даних,отримувати підтвердження бронювання електронною поштою,переглядати відгуки, переглядати контакти готелю, та можливість зареєстуватись.

**Персонал-STAFF** можуть переглядати доступні номери, обирати дати,фільтрувати, сортувати, переглядати всі бронювання, фільтрувати за типом бронювання, створювати/редагувати/скасовувати бронювання, переглядати відгуки та модерувати їх, переглядати сторінку профілю та змінювати інформацію про себе та пароль.

**Персонал-ADMIN** можуть переглядати доступні номери, створювати/редагувати/видаляти всі номери, обирати дати, переглядати всі бронювання, створювати/редагувати/скасовувати всі бронювання, переглядати всіх користувачів, створювати/редагувати/видаляти користувачів, дивитись статистику по всіх компонентах, переглядати відгуки, модерувати відгуки, переглядати сторінку профілю та змінювати інформацію про себе та пароль 

## Технологічний стек та архітектура системи 

Система побудована за **клієнт-серверною архітектурою**.

* **Backend:** Python, Flask, Flask-Smorest, Marshmallow + PostgreSQL
* **Frontend:** JavaScript, HTML, CSS 
* **Тестування:** pytest з покриттям (codecov) > 86%
* **Документація:** Swagger UI + Markdown documentation, OpenAPI 3.0
* **Контейнеризація:** Docker, Docker Compose
